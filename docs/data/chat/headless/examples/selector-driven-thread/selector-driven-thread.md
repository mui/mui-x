# Selector-driven thread

<p class="description">Render large custom threads efficiently with ids at the list level and row-level message subscriptions.</p>

This recipe focuses on:

- `useMessageIds()`
- `useMessage(id)`
- `useConversations()`
- `useConversation(id)`

The parent updates a single controlled message so the example can show why selector-driven rendering matters.
